COMMANDS REFERENCE

The bl command provides a comprehensive interface to The Backlogs system. All operations on phases, milestones, epics, and tasks are conducted through this procedural interface. Each command is documented with its full parameter set and expected behavioral contract.

[NOTE]

bl is a registered alias for backlog. The two commands are interchangeable. The tasks alias is also supported for backward compatibility with legacy installations.

§2.1 Core Task Operations

The following commands form the primary operational interface for task lifecycle management. Each command operates within the context of the current working directory's .backlog/ hierarchy.

Command Function
bl list Filter and view tasks. Supports --available, --progress, --json, --bugs, --ideas.
bl grab Automatically claim the next available task from the critical path. See BL-CMD-GRAB.
bl tree Full hierarchical view of the backlog structure. Supports --depth, --details, --unfinished.
bl cycle Complete current task and auto-claim next. Combines done + grab.
bl done Mark the current task as complete. Shows newly unblocked work.
bl show Detailed task information. Uses current context if no ID provided.
bl blocked Mark current task as blocked. Requires --reason.
bl why Explain dependency readiness for a given task.

§2.2 Example Output

A typical invocation of bl list --available produces the following output: 

$ bl list --available

  Available Tasks (4)
  ───────────────────────────────────────────────
  P1.M2.E1.T042  Implement retry logic     high    2h   0 deps
  P1.M2.E1.T043  Add timeout config        medium  1h   0 deps
  P1.M2.E2.T001  Setup test harness        high    3h   0 deps
  P1.M3.E1.T001  Design API schema         high    4h   0 deps

  Critical path: P1.M2.E1.T042 → P1.M2.E1.T043 → P1.M2.E2.T001

§2.3 Workflow Shortcuts

For common operational patterns, the following shortcut commands are provided. These combine multiple operations into single procedural steps.

Command Function
bl grab Auto-claim next work (--single, --multi, sibling batching)
bl cycle done + auto-claim next
bl work Set, show, or clear working context
bl handoff Transfer to another agent (--to, --notes)
bl skip Skip current task without completing
[TIP]

Use bl cycle for the fastest claim-work-complete loop. It is equivalent to running bl done followed by bl grab, but handles edge cases around sibling task batching automatically.
