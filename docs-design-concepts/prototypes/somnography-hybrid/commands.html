Commands

The Backlogs provides the following command set for navigating the task hierarchy. Each command operates within the context of a .backlog/ project directory containing YAML indexes and Markdown task files.

Core Task Operations

The primary command loop follows a claim-work-complete pattern. Tasks are selected from the dependency graph based on critical path analysis and priority scoring.

Command Description Status
backlog list Filter and view tasks with flags like --available, --progress, --json Stable
backlog tree Full hierarchical view with --depth and --details options Stable
backlog show [ID] Detailed task information; uses working context if no ID given Stable
backlog grab Auto-claim next available task from the critical path See CH2
backlog done Complete current task, display newly unblocked work Stable
backlog cycle Combined done + grab in one operation See notes
backlog sync Recalculate statistics and critical path analysis Stable
Annotation

All commands operate relative to the nearest .backlog/ directory. The engine walks upward from the current working directory to locate the project root. If multiple backlog directories exist in the path, the nearest ancestor is used.

Workflow Shortcuts

For continuous work sessions, the workflow shortcuts eliminate the claim-inspect-work-complete ceremony:

# The standard loop
backlog grab            # pick up next task
backlog show            # read the details
# ... do the work ...
backlog done            # complete it

# Or simply:
backlog cycle           # done + grab in one step

The grab command uses a sophisticated selection algorithm that considers dependency readiness, critical path position, priority weighting, and agent affinity. See CH2: Grab for detailed documentation.

Additional workflow commands include blocked for marking impediments, skip for deferring tasks, and handoff for transferring work between agents.
